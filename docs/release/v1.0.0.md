# Release v1.0.0

**v1.0.0** is the first **stable local / demo / research platform** release of this repository: a **Kafka + Spark + PostGIS** correlator for **open-data experimentation** and **engineering evaluation** of smoke-risk–related signals. It is suitable for **reproducible local workflows**, fixture-driven demos, and operator inspection via SQL and Grafana.

## What v1.0.0 is for

- **Local reproducible experimentation** with FIRMS-style hotspots, OpenAQ-style measurements, optional wind / grid weather, and census-linked geometries.
- **Engineering evaluation hooks** (risk observations, dispersion–AQ comparisons, calibration labels) as **triage scaffolding**, not validated science.
- **Operational transparency**: SQL-first alerting candidates, runbooks, optional exports, and Grafana tables/maps.

## What v1.0.0 explicitly does **not** claim

- **Not public-health validated** — scores and calibration outputs are **not** clinical or epidemiological advice.
- **Not regulatory dispersion or compliance AQ modeling** — plume corridors, Gaussian proxy, and lag summaries are **engineering heuristics**.
- **Not operational emergency guidance** — do not use this stack as an authoritative emergency information system without external validation and governance.

## Included capabilities (summary)

- Ingestion / normalization paths through Phase 12 calibration surfaces; Phase 13 CI/export/release scaffolding; **Phase 14** migration integrity (`013_phase14_canonical_alert_function.sql`), **`make db-doctor`**, **`make repair-alert-function`**, **`make release-manifest`**, optional **`make release-fresh-volume-test`** on an **isolated Compose project**.
- Risk models **v1–v5**, plume **wind_v1** / **wind_grid_v2**, dispersion proxy **gaussian_v0** — all scoped as **non-regulatory** correlations in docs and manifests.

## Data sources

Same as prior milestones: optional live FIRMS / OpenAQ / NWS with explicit bounding and keys; Census TIGER for **`make db-bootstrap`**; **synthetic minimal census** for **`make db-bootstrap-minimal`** / CI.

## Validation (maintainers)

```bash
uv sync --extra dev
uv run ruff check src tests
uv run pytest -q
SMOKE_NO_COMPOSE=1 bash scripts/smoke_test.sh
make version
make release-check
```

With Compose and migrations applied:

```bash
make db-doctor
make repair-alert-function   # only when legacy volumes show fn_alert_candidates overload drift
CALIBRATION_EXPORT_DRY_RUN=1 uv run python -m wildfire_smoke.export_calibration
```

See **`docs/release/v1.0.0-checklist.md`** before tagging.
